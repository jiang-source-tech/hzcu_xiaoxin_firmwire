# 小新低功耗贪吃蛇屏保背景设计

## 目标

低功耗屏保界面保留现有时间、日期、同步状态和唤醒提示，同时把纯黑加圆弧的背景升级为类似 GitHub contribution snake 的循环动画。动画作为背景存在，不替代时间信息，也不进入宠物情绪 GIF 系统。

成功标准：

- 进入低功耗屏保后，第一眼仍能读到大号 `HH:MM`。
- 背景出现暗色贡献格子和一条低亮度小蛇，形成持续循环的屏保感。
- 小蛇不会遮挡中心时间、日期、同步状态和底部 `POWER` 唤醒提示。
- 不新增 GIF、WebP 或大体积图片资源，避免增加 Flash 和 GIF 解码压力。
- 不创建数百个 LVGL 子对象；背景默认由一个自绘 LVGL 对象完成。
- 动画刷新频率保持低，适合低功耗屏保。

## 体验设计

屏幕背景使用近黑底色。底层绘制一组小方格，视觉参考 GitHub 贡献图：大部分格子是暗绿蓝色，少量格子有不同亮度层级。小蛇由连续的亮色方格组成，蛇头稍亮，身体逐步变暗。

小蛇沿确定性的预设网格路径移动，而不是随机游走。路径绕开中心时间阅读区和底部提示区，到达末尾后回到起点，形成循环屏保。

前景信息继续使用当前屏保布局：

- 中心大号时间。
- 日期文本。
- 同步状态点和同步文本。
- 底部唤醒提示。
- 现有内外圆弧保留，但整体调暗，避免和小蛇背景、中心时间抢层级。

## 技术选择

默认采用单个自绘 LVGL 背景对象，而不是每个格子一个 `lv_obj_t`。

当前配置为 LVGL 9.4，`CONFIG_LV_USE_CANVAS=y`，PSRAM 已开启。但全屏 RGB565 canvas 需要约 `412 * 412 * 2 = 339 KB` 的像素缓冲，不适合作为默认低功耗屏保方案。每格一个 LVGL 对象又可能产生 300 到 600 个对象，带来明显堆内存和 style 开销。

因此默认方案为：

- 创建一个 `low_power_clock_snake_bg_`，大小覆盖屏幕。
- 给该对象注册 `LV_EVENT_DRAW_MAIN` 回调。
- 回调中使用 `lv_event_get_layer()` 和 `lv_draw_rect()` 绘制格子和蛇。
- 每次动画 tick 只更新 `low_power_clock_snake_tick_` 并调用 `lv_obj_invalidate(low_power_clock_snake_bg_)`。

该方案的 LVGL 对象增量为 1 个，运行内存主要是少量静态路径数据和局部 draw 描述符。

若 draw 回调在实机上出现兼容问题，降级策略是创建一个小尺寸 indexed canvas 或稀疏对象方案，但不能回到 400+ 格子对象的默认实现。

## 网格参数

固定使用稀疏网格，先按可读性和内存稳定性保守实现：

- `cell_size = 12`
- `cell_gap = 6`
- `pitch = 18`
- `origin_x = 17`
- `origin_y = 17`
- 最大列数约 22，最大行数约 22。

初始化或绘制时按圆形屏幕裁剪格子。以屏幕中心 `(206, 206)` 为圆心，使用可视半径 `198`：

```c
cell_cx = x + cell_size / 2;
cell_cy = y + cell_size / 2;
inside_circle = (cell_cx - 206) * (cell_cx - 206) +
                (cell_cy - 206) * (cell_cy - 206) <= 198 * 198;
```

圆外格子不绘制。这样不依赖 LVGL 自动圆形裁剪，也不会让四角格子进入实际屏保画面。

安全区：

- 中心时间安全区：`x = 82..330, y = 135..245`。
- 日期和同步状态软安全区：`x = 86..326, y = 246..292`，允许暗背景格子，但蛇身不进入。
- 底部提示安全区：`y >= 326`。

背景基础格子可以出现在日期软安全区，但蛇头和蛇身不能进入这些文字区域。

## 路径算法

路径由网格坐标组成，初始化时生成一次并保存在静态数组或固定大小成员数组里。

生成规则：

1. 遍历所有网格格位。
2. 跳过圆外格子。
3. 跳过中心时间安全区、日期同步软安全区和底部提示安全区。
4. 只保留剩余格子作为候选路径。
5. 采用按行往返的 typewriter 路径：偶数行从左到右，奇数行从右到左。
6. 行与行之间天然通过相邻端点连接；如果因为安全区导致断裂，允许一次性跳跃到下一段起点，跳跃过程中不额外绘制连接线。

蛇参数：

- `snake_length = 9`。
- `snake_head_index = low_power_clock_snake_tick_ % path_count`。
- 第 `i` 段身体使用 `path[(snake_head_index + path_count - i) % path_count]`。
- `i = 0` 是蛇头，`i = 1..8` 是身体。

循环策略：

- 路径数组本身按环处理。
- 到达末尾后直接取模回到起点。
- 因为这是屏保背景，不要求真实贪吃蛇的连续物理运动；允许路径段切换时有轻微跳转，但不能跳进文字安全区。

## 绘制策略

每次 `LV_EVENT_DRAW_MAIN`：

1. 先绘制所有可见基础格子。
2. 再绘制蛇身格子。
3. 最后绘制蛇头格子。

基础格子颜色使用确定性 hash，避免保存每个格子的随机状态：

```c
level = (col * 17 + row * 31 + 7) % 5;
```

根据 `level` 选择极暗到中暗的背景颜色。蛇身使用固定长度数组计算，不需要修改每个格子的 LVGL style。

性能预期：

- 每秒 invalidate 一个背景对象。
- 每次 draw 绘制约 250 到 330 个基础矩形，加 9 个蛇矩形。
- 不分配新内存，不重建对象。
- 当前 LVGL 刷新周期是 33ms，但该背景只由低功耗时钟 timer 每 1 秒触发一次动画更新。

## 颜色与亮度

极暗色在 12% 背光下可能不可见，因此第一版不使用过低的基础亮度。建议色板：

- 背景：`0x000000`。
- 基础格子 0：`0x0B1A13`，`LV_OPA_50`。
- 基础格子 1：`0x10261B`，`LV_OPA_55`。
- 基础格子 2：`0x143322`，`LV_OPA_60`。
- 活跃暗格：`0x1B4D33`，`LV_OPA_70`。
- 蛇身暗段：`0x24734A`，`LV_OPA_75`。
- 蛇身亮段：`0x2F9E5D`，`LV_OPA_85`。
- 蛇头：`0x56D364`，`LV_OPA_95`。

实机偏亮时，优先降低蛇身和基础格子的 opacity；实机偏暗时，优先提高基础格子 opacity 到 `LV_OPA_70`，不提高低功耗屏幕整体亮度，避免影响省电策略。

## 圆弧调整

现有圆弧保留但调暗：

- 外弧 main：`0x071015`，`LV_OPA_35`，宽度 2。
- 外弧 indicator：`0x123428`，`LV_OPA_35`，宽度 2。
- 内弧 main：`0x050A0C`，`LV_OPA_20`，宽度 3。
- 内弧 indicator：`0x1C6B4A`，`LV_OPA_55`，宽度 4。

内弧继续旋转，但不再使用当前高亮青色 `0x26D9FF` 的满不透明效果。

## 错误处理

- 如果 `low_power_clock_snake_bg_` 创建失败，继续显示现有时间、日期、同步状态、提示和圆弧。
- 如果路径生成后 `path_count < snake_length + 4`，禁用小蛇绘制，只绘制基础格子。
- 如果低功耗屏保隐藏，沿用现有 `StopLowPowerClockRefreshTimer()` 停止动画驱动。
- 如果时间无效，继续沿用当前 `--:--` 行为，小蛇背景仍可播放。

## 测试策略

更新或新增低功耗屏保路径测试，覆盖：

- 低功耗屏保仍创建时间、日期、同步状态和提示控件。
- 存在 `low_power_clock_snake_bg_` 或等价的单对象背景初始化路径。
- 不存在每格创建 `lv_obj_create()` 的实现模式。
- snake 背景对象创建早于前景时间标签，保证层级在后。
- 动画刷新路径更新 snake tick 并 invalidate 背景对象。
- 圆屏裁剪使用中心距离判断。
- 路径生成显式跳过中心时间安全区和底部提示安全区。
- 不依赖新增 GIF 或图片资源。

继续运行相关 C 单元测试和 Python 路径测试。实现后至少运行：

- `pytest tests/xiaoxin_low_power_clock_visual_path_test.py`
- 相关 `xiaoxin_low_power_clock_model` 测试
- `idf.py build`

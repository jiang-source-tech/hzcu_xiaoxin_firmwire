# 小芯设置页动态亮度滑杆设计

> 日期：2026-06-23  
> 范围：Waveshare ESP32-S3 Touch LCD 1.46 设置页中的「亮度」单项编辑页

## 背景

当前亮度设置页只显示 `30  70  100` 三个数字，并在进入页面时直接套用中间预设。这个界面缺少可见的当前值、没有连续调节反馈，也不符合圆形小屏上“少信息、强反馈”的设置体验。

底层背光能力已经存在：`Backlight::SetBrightness(uint8_t brightness, bool permanent)` 支持 0 到 100 的百分比，并在 `permanent == true` 时写入 `Settings("display")` 的 `brightness`，重启后由 `RestoreBrightness()` 恢复。因此本次不新增存储路径，只改善亮度页的 UI 和触摸交互。

## 目标

- 把亮度页从三档预设改为连续的 10 到 100 百分比调节。
- 页面打开时显示当前背光百分比，而不是强制切到 70。
- 拖动时实时预览屏幕亮度变化。
- 松手时保存最终亮度，重启后保持。
- 视觉上从“数字贴底”改为更清晰的圆屏友好布局。

## 非目标

- 不改 Wi-Fi、关于、省电等其他设置项。
- 不新增独立 NVS 键或绕过 `Backlight` API。
- 不引入复杂圆形旋钮；第一版使用横向滑杆，降低触摸误差。
- 不改变 BOOT 长按进入设置页、BOOT 短按返回/关闭设置页的现有语义。

## 交互设计

亮度页结构：

```text
        亮度

        70%

   ━━━━━━━━━━━━━
      低     高

     拖动调节 · BOOT 返回
```

用户点击设置列表中的「亮度」后进入亮度编辑页：

- 页面读取 `Board::GetInstance().GetBacklight()->brightness()` 作为初始值。
- 中央显示当前值，例如 `70%`。
- 滑杆区域支持点击和水平拖动。
- 触点在滑杆左端映射为 10，右端映射为 100，超出范围时夹紧。
- 按下或拖动时调用 `Backlight::SetBrightness(value, false)` 做实时预览，避免每一帧都写 NVS。
- 松手时调用 `Backlight::SetBrightness(value, true)` 保存最终值。
- 亮度页中再次点击非滑杆区域不返回；返回仍沿用现有单项页触摸返回逻辑或 BOOT 短按返回设置列表。

## 视觉设计

继续沿用当前设置 overlay 的深色科技风，但亮度页减少噪声：

- 标题：顶部 `亮度`。
- 数值：中间大号百分比文本，使用主文本色。
- 滑杆轨道：圆角深色底轨。
- 滑杆已填充段：使用设置页蓝色强调色。
- 滑块：小圆点或短圆角块，跟随百分比移动。
- 底部提示：`拖动调节 · BOOT 返回`。

在 1.46 寸圆形屏上，滑杆有效宽度控制在设置面板内，避免靠近圆屏边缘的触摸死角。

## 架构与数据流

新增轻量模型函数，放在现有 `xiaoxin_settings_model` 中：

- `xiaoxin_settings_brightness_from_x(x, left, width)`：把触摸横坐标映射到 10 到 100。
- 继续使用 `xiaoxin_settings_clamp_percent()` 做边界保护。

显示层新增亮度页对象成员：

- 百分比标签。
- 滑杆轨道。
- 滑杆填充段。
- 滑块。
- 当前亮度缓存值。
- 是否正在拖动亮度滑杆。

数据流：

```text
触摸 x 坐标
  -> xiaoxin_settings_brightness_from_x()
  -> 更新亮度页 UI
  -> Backlight::SetBrightness(value, false) 实时预览
  -> 松手 Backlight::SetBrightness(value, true) 保存
```

## 错误处理和边界

- `GetBacklight()` 返回空：页面仍显示默认 75%，但不会崩溃；触摸只更新 UI。
- 触摸横坐标越界：夹紧到 10 或 100。
- 当前亮度为 0：页面显示时提升为 10，避免把屏幕调到全黑，也和 `RestoreBrightness()` 的安全默认策略保持一致。
- 快速拖动：每次触摸轮询只做轻量 LVGL 尺寸/位置更新和背光目标更新。

## 测试策略

- C 模型测试：
  - 百分比 clamp 保持现有覆盖。
  - 新增 x 坐标到亮度映射测试：左端、右端、中点、越界、宽度为 0。
- Python source-path 测试：
  - 亮度页不再暴露 `30 / 70 / 100` 三档预设。
  - 亮度页创建百分比标签、轨道、填充段和滑块。
  - 触摸处理在亮度页调用动态映射函数。
  - 拖动预览使用 `SetBrightness(value, false)`，松手保存使用 `SetBrightness(value, true)`。
- 构建验证：
  - 至少运行设置模型 C 测试。
  - 运行设置页路径测试。

## 验收标准

- 进入亮度页不会自动跳到 70。
- 用户可以通过滑动连续调节亮度。
- 页面上的百分比和滑杆位置随触摸同步变化。
- 松手后的亮度会持久化。
- BOOT 返回行为仍可用。
- 设置页打开时外层卡片分页手势仍被抑制。

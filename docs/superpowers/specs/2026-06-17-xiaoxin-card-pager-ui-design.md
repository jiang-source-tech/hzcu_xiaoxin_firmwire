# 小芯卡片分页 UI 设计规格

> 状态：已确认 | 日期：2026-06-17 | 设备：Waveshare 1.46" 圆屏 ESP32-S3

## 1. 设计方向

**审美路线**：极简科技 (Minimalist Tech)
**布局策略**：毛玻璃卡片层叠 (Glass-morphism Cards)
**配色基调**：深色底 + 蓝色强调 (Blue Accent)

整体追求高端智能手表的克制感——深色背景、细线分隔、蓝调边框、通过透明度和字号建立信息层级。卡片页与 Home 页的白色宠物背景形成"日夜切换"的戏剧感。

## 2. 色彩系统

## 3. 布局规格

### 3.1 通知页（下滑进入）

```
┌──────────────────────────┐
│      [下拉指示条 34×3]    │  ← 距顶约 24px
│         🔔 通知           │  ← 页面标题，间距 8px + 14px
│                          │
│  ┌─────────────────────┐ │  ← 卡片 1: 195px 宽
│  │ ●  ⚡ 低电量        ›│ │     背景 opacity 最高
│  │    请尽快充电        │ │     状态点红色+光晕
│  └─────────────────────┘ │
│  ┌─────────────────────┐ │  ← 卡片 2: 间距 8px
│  │ ●  📋 课程变更      ›│ │     背景 opacity 中等
│  │    教室待确认        │ │     状态点橙色+光晕
│  └─────────────────────┘ │
│  ┌─────────────────────┐ │  ← 卡片 3: 间距 8px
│  │ ●  💬 聊天提醒      ›│ │     背景 opacity 最淡
│  │    新的小芯消息      │ │     状态点蓝色+光晕
│  └─────────────────────┘ │
│      [Home 指示条 56×3]  │  ← 距底约 8px
└──────────────────────────┘
```

**布局参数**：
- 卡片宽度：195px（在 280px 可视区域内居中）
- 卡片内边距：10px 上/下, 14px 左/右
- 状态点直径：8px，圆角 50%
- 状态点与文字间距：8px
- 卡片标题行高：1.3
- 卡片正文行高：1.2
- 卡片间距：8px
- 仅显示 3 条通知（与现有 `k_card_item_label_count` 逻辑一致）

### 3.2 总览页（上滑进入）

```
┌──────────────────────────┐
│      [Home 指示条 56×3]  │  ← 距顶约 8px (指示条在顶部)
│                          │
│  📚  下一节课           ›│  ← 图标 28×28 + 间距 10px
│      高等数学 10:10      │     圆角 8px 图标背景
│  ───────── 分隔线 ───────│  ← rgba(74,158,255, 0.08)
│  🗺️  校园导航           ›│
│      常用地点入口        │
│  ───────── 分隔线 ───────│
│  🌤️  天气               ›│
│      多云 26°C           │
│  ───────── 分隔线 ───────│
│  ✅  今日待办           ›│
│      还有 2 项           │
│                          │
│         📋 总览           │  ← 页面标题在底部
│      [底部指示条 34×3]    │  ← 距底约 6px
└──────────────────────────┘
```

**布局参数**：
- 行高：图标 28×28px + 10px 间距 + 文字区域
- 分隔线：180px 宽, 0.5px 高
- 图标背景：28×28px, 圆角 8px，颜色跟随分类
- 顶部/底部指示条位置互换（总览页从底部滑入）

### 3.3 标题位置逻辑

- **通知页**：标题在卡片上方（顶部区域）
- **总览页**：标题在卡片下方（底部区域）
- 原因：通知页从顶部下滑，内容从上往下展开；总览页从底部上滑，内容从下往上展开。标题跟随滑入方向。

## 4. 动画规范

### 4.1 页面切换动画（已有，保持）

- 跟手拖动：卡片 layer 偏移 = `offset_y / 4`
- 松手吸附：`ease_out` 曲线, 210ms
- 阈值：屏幕高度 20%
- 最大拖拽：屏幕高度 25%
- 拖动时透明度：`LV_OPA_80`
- 静止时透明度：`LV_OPA_COVER`

### 4.2 新增：卡片渐入动画

页面吸附到位后，卡片逐张淡入：

- 卡片 1：延迟 0ms，淡入 120ms
- 卡片 2：延迟 50ms，淡入 120ms
- 卡片 3：延迟 100ms，淡入 120ms
- 总览页各条目同理，从上到下 stagger 50ms

### 4.3 新增：状态点呼吸动画（可选，低优先级）

- 仅红色紧急状态点做 pulsing：opacity 50%↔100%，周期 1.2s
- 其他颜色静态
- 实现方式：LVGL anim on opacity

## 5. 字体

- LVGL 默认内置字体（`LV_FONT_DEFAULT`）
- 标题可尝试使用稍微加粗的变体
- 正文使用默认大小
- 未来可选：为中文标题制作小尺寸 bitmap 字体以获得更好渲染效果

## 6. LVGL 实现要点

### 6.1 卡片 layer 结构

```
card_layer_ (lv_obj, 全屏, 透明背景)
├── pull_indicator_ (lv_obj, 3px 高横条)
├── card_title_label_ (lv_label, 页面标题)
├── card_item_containers_[0..2] (lv_obj, 圆角矩形卡片)
│   ├── status_dot_ (lv_obj, 8px 圆形)
│   ├── title_label_ (lv_label, 卡片标题)
│   ├── body_label_ (lv_label, 卡片正文)
│   └── arrow_label_ (lv_label, ›)
├── separator_lines_[0..2] (lv_obj, 0.5px 横线, 仅总览页)
├── icon_bg_[0..3] (lv_obj, 28×28 圆角方块, 仅总览页)
└── home_indicator_ (lv_obj, 3px 高横条)
```

### 6.2 关键 LVGL API

- 背景色带透明度：`lv_color_hex(0x0c0f16)` + `lv_obj_set_style_bg_opa(obj, LV_OPA_80, 0)`
- 卡片边框：`lv_obj_set_style_border_color(obj, lv_color_hex(0x4a9eff), 0)` + `lv_obj_set_style_border_opa(obj, LV_OPA_14, 0)` + `lv_obj_set_style_border_width(obj, 1, 0)`
- 圆角：`lv_obj_set_style_radius(obj, 12, 0)`
- 状态点光晕：使用 `lv_obj_set_style_shadow_color` + `lv_obj_set_style_shadow_width` + `lv_obj_set_style_shadow_spread`
- 卡片背景渐变：LVGL 不支持复杂渐变，但可通过多层叠加或预渲染背景图实现；第一版使用纯色半透明背景即可

### 6.3 性能考虑

- 卡片页面的所有视觉元素仅在页面可见时创建/更新（已有 `LV_OBJ_FLAG_HIDDEN` 机制）
- Home 页不渲染任何卡片 UI
- 状态点呼吸动画仅在通知页激活时运行
- 不使用真正的 backdrop blur（ESP32 性能不足），用半透明背景替代

## 7. 页面差异化

| 维度 | 通知页 | 总览页 |
|------|--------|--------|
| 进入方向 | 从顶部下滑 | 从底部上滑 |
| 内容基调 | 紧急、需要关注 | 稳定、日常浏览 |
| 卡片样式 | 层叠毛玻璃，透明度递减 | 图标+细线分隔，透明度均匀 |
| 状态点 | 有（红/橙/蓝+光晕） | 无（图标颜色区分） |
| 标题位置 | 卡片上方 | 卡片下方 |
| 指示条 | 顶部下拉条 | 底部上拉条 |
| 最大条目 | 3 条 | 4 条 |

## 8. 与现有代码的集成

修改集中在 `InitializeCardPagerLayer()` 和 `RenderCardPage()` 两个函数：

1. `InitializeCardPagerLayer()`：重新创建 card_layer_ 的子对象，包含卡片容器、状态点、图标等新元素
2. `RenderCardPage()`：按页面类型（通知/总览）分别渲染不同布局
3. 新增 `ApplyCardEntryAnimation()`：页面吸附后播放卡片渐入动画
4. 删除现有的 `card_item_labels_[]` 纯文字方案，替换为结构化卡片容器

# 小新低功耗贪吃蛇随机游走与渐变色设计

## 目标

把低功耗时钟背景里的贪吃蛇从固定扫描路径改为实时随机游走，并把蛇身颜色从几档固定绿色改为沿身体方向的渐变色。它仍然是低功耗时钟的背景动效，不变成可玩的贪吃蛇游戏。

成功标准：

- 蛇每个动画 tick 都根据随机方向移动，看起来不是沿固定路径循环。
- 蛇保持连续，不瞬移、不断裂，不进入圆屏外、底部提示避让区和当前蛇身占用格。
- 如果随机方向不可走，会自动选择其他可走方向；如果被短暂围住，会重置到一个可用起点。
- 蛇身从头到尾呈现清晰渐变，蛇头最亮，身体逐步过渡到偏蓝绿和更低透明度的尾部。
- 背景仍由单个自绘 LVGL 对象完成，不增加 GIF、图片资源或大量 LVGL 子对象。

## 体验设计

蛇在 22x22 的稀疏网格里移动。每次移动优先从三个候选方向中选择：直行、左转、右转。候选顺序由轻量随机数打散，因此蛇会有自然的随机转向；同时直行仍有机会被优先选中，避免画面过于抖动。

蛇不允许反向穿过自己的身体。下一格必须满足：

- 在圆形可视区域内。
- 不进入底部提示和日期同步软避让区域。
- 不撞到当前蛇身，尾部即将离开的格子可视为可用。

如果三个候选方向都不可走，再尝试四个方向中的任意可走方向。如果仍然没有可走方向，说明蛇被局部困住，下一 tick 将从可用区域中选一个随机起点重新生成短蛇。这种重置应很少发生，并且不影响前景时间显示。

## 技术设计

继续复用现有低功耗时钟背景绘制结构：

- `low_power_clock_snake_bg_` 仍是唯一的蛇背景 LVGL 对象。
- `LV_EVENT_DRAW_MAIN` 回调里先绘制基础格子，再绘制蛇身。
- 动画 tick 仍由低功耗时钟 timer 驱动，并 invalidate 背景对象。

新增或调整的状态：

- 保存蛇身数组，长度为 `k_low_power_snake_length`，数组第 0 项为蛇头。
- 保存当前移动方向。
- 保存随机数状态，用确定性的轻量 PRNG 驱动方向选择。
- 初始化或重置时，从所有可用格中随机选择起点，再选择一个可行方向生成连续短蛇。

随机数不需要加密强度。可使用基于 tick、当前时间和对象状态混合的 32 位 xorshift 或类似算法。测试只验证存在随机状态与候选方向选择，不要求固定随机序列。

## 渐变色设计

蛇身颜色按 `body_index` 计算，而不是硬编码三段颜色。

建议渐变：

- 头部：`0x7CFFB2`，高透明度。
- 前段身体：向 `0x32D6A0` 过渡。
- 后段身体：向 `0x1E7EA3` 过渡。
- 尾部：更低透明度，保持背景感。

实现上增加一个小的颜色插值 helper，从起始 RGB 到结束 RGB 线性插值，再配合透明度插值。这样蛇长变化时也能自然适配。

## 错误处理

- 如果蛇背景对象不存在，低功耗时钟继续显示前景时间、日期、同步状态和提示。
- 如果随机初始化找不到足够可用格，禁用蛇身绘制，只保留基础背景格。
- 如果运行中被困住，重置蛇状态，不崩溃、不阻塞 UI。
- 如果可用方向数量为 0，下一次动画 tick 继续尝试重置。

## 测试策略

先用测试约束行为，再实现：

- 更新 `tests/xiaoxin_low_power_clock_visual_path_test.py`，确认不再使用固定 `BuildLowPowerSnakePath()` 循环路径。
- 测试存在蛇身状态、方向状态和随机状态。
- 测试存在随机方向候选选择或等价 helper。
- 测试下一步会检查圆屏裁剪、蛇避让区和自碰撞。
- 测试蛇绘制使用颜色插值或渐变 helper，而不是固定三档绿色。
- 保留现有单对象背景、背景先于前景标签创建、动画 tick invalidate 背景对象等断言。

实现后至少运行：

- `pytest tests/xiaoxin_low_power_clock_visual_path_test.py`
- 如环境可用，再运行相关 C 模型测试和 `idf.py build`

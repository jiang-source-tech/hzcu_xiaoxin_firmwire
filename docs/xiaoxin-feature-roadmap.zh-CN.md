# 小芯后续功能拓展路线图

> 日期：2026-06-24
> 范围：Waveshare ESP32-S3 Touch LCD 1.46 小芯/泡泡宠物固件
> 目标：基于现有宠物首页、触摸卡片分页、通知中心、总览页、电量/网络状态浮层，规划后续可持续扩展方向。

## 0. 维护规则

本文档不仅记录未来功能方向，也必须持续记录每个方向的真实实现状态。后续新增或调整需求时，请同步补充：

- 已实现：已经合入代码、可运行或已有测试覆盖的能力。
- 部分实现：已有接口、模型、UI 或 mock，但尚未完整接入真实数据源或真实业务流程。
- 未实现：仍停留在设计、预留类型、文档说明或计划中的能力。
- 下一步：把该方向推进到下一个可验收状态所需的最小工作。

状态说明建议使用：

- `已完成`：该方向的核心用户路径已经可用，并有基本验证。
- `部分完成`：基础结构或局部事件已完成，但仍缺真实数据、调度、过期策略或端到端接入。
- `未开始`：只有规划，尚无明确代码实现。

## 1. 当前基础

当前项目已经具备较完整的交互骨架：

- 宠物首页：泡泡宠物 GIF 动画、触摸/摇晃/服务端情绪/设备事件触发，不同设备状态映射已通过独立情绪模型收束。
- 卡片分页：Home、Notifications、Overview 三页，支持上下滑切换和吸附动画。
- 通知中心：真实通知事件仓库、通知卡片、纵向浏览、左滑单条清理、全部清理、空状态。
- 总览页：已从静态示例数组迁到 `xiaoxin_overview_model` 快照模型，支持时间日期、天气、课程、待办和设备状态的真实数据接入入口。
- 系统浮层：WiFi 状态、电量状态、低电量警示色。
- 设置能力：BOOT 长按设置页、亮度滑条、Wi-Fi 重新配网、关于页和省电开关已经落地。
- 低功耗能力：电量 ADC 读取、长按关机、深睡唤醒、省电调度器、低功耗时钟和 POWER 唤醒。
- 测试基础：卡片分页、通知事件、总览模型、宠物情绪、设置模型、低功耗时钟、电量换算、系统浮层、电源控制等本地测试。

后续功能应优先复用这些能力，避免另起一套 UI 或状态机。

## 2. 推荐优先级

### P0：真实通知中心

当前状态：`部分完成`。

目标：让通知页从静态样板变成真实事件入口。

已实现：

- 通知中心已从静态样板改为事件注入模型，支持 `upsert`、移除、单条清理、全部清理和优先级排序。
- 低电量通知已接入板级状态刷新逻辑，低于阈值时注入，恢复后移除。
- WiFi 断开和配网中状态已接入通知仓库，连接恢复后移除。
- 语音错误状态已通过状态栏错误分支注入语音识别失败通知。
- 上课提醒已有 `xiaoxin_course_reminder_t` helper，可把课程记录转换为通知事件。
- 普通聊天回复已不再作为通知中心事件方向，避免和对话内容重复。
- 已有本地测试覆盖通知事件注入、upsert、排序、移除、清理、课程提醒 helper 和聊天回复忽略。

未实现或未完整实现：

- `ttl_ms` 目前只是事件字段和默认值，尚未看到自动过期扫描或定时移除逻辑。
- OTA 更新事件已有类型和默认文案，但尚未从 OTA 检查、下载、升级失败等真实流程注入通知中心。
- 课程提醒只有可注入 helper，尚未接真实课表源、调度器、同一节课去重和本地提醒策略。
- 定时提醒尚未看到独立数据源和调度入口。
- 通知清理和外部状态再变化后的重新出现策略已有方向，但还需要在更多真实事件源上验证。

下一步：

- 为 `ttl_ms` 增加到期清理机制，并覆盖语音识别失败这类短期通知。
- 将 OTA 检测、下载中、失败等状态转换为 `XIAOXIN_NOTIFICATION_EVENT_OTA_UPDATE`。
- 接入第一版课程/提醒数据源，完成“真实课表或提醒记录 -> 本地调度 -> 通知事件”的闭环。

建议接入的事件：

- 低电量。
- WiFi 断开或配网中。
- OTA 发现新版本、下载中、升级失败。
- 语音识别失败。
- 需要离开对话页后仍提醒用户的业务消息。
- 定时提醒或课程提醒。

设计要点：

- 保持通知中心内部有一套统一事件模型。
- 同一类状态型通知使用 upsert，避免重复堆积。
- 临时型通知可使用 TTL，超时后自动消失。
- 通知清理只影响当前通知仓库，不破坏外部真实状态。

说明：项目中已有 `docs/xiaoxin-real-notification-center.zh-CN.md`，可以作为该方向的具体实现说明继续维护。

### P1：动态总览页

当前状态：`部分完成`。

目标：让 Overview 页成为用户抬手就能看的日常摘要。

当前基础：

- Overview UI 已保留四张卡片形态，并由 `xiaoxin_overview_snapshot_t` 提供渲染数据。
- 天气、课程、待办尚未接真实服务，当前显示未同步、未配置或暂无状态。
- 设备状态卡片只显示定性电量状态，不显示 ADC 估算百分比。

已实现：

- 已有 `xiaoxin_overview_model` 快照模型，UI 层通过 `xiaoxin_overview_snapshot_t` 渲染 Overview。
- Overview 已包含天气、下一节课、今日待办、设备状态四个固定摘要项。
- 时间、日期、星期可从本机时间构建；时间无效时显示未同步。
- 设备状态已接入网络连接状态和电量读取结果。
- 天气、课程、待办已预留注入字段，并能根据未联网、未配置、无数据等状态显示对应空态。
- 已有本地测试覆盖离线默认值、联网未配置、富数据注入、未知电量、电量状态分档和快照自持有文本。

未实现或未完整实现：

- 板级 `BuildOverviewState()` 当前只填入时间、网络和电量；天气、课程、待办仍被写为未配置或空数据。
- 尚未接入服务端天气摘要同步。
- 尚未接入服务端或配置端课程摘要同步。
- 尚未接入今日待办数量和摘要同步。
- 设备状态仍偏轻量，尚未展示设备温度、可用内存等更完整摘要。
- 电量显示仍以定性状态为主，未展示可信百分比或校准后的电量信息。

下一步：

- 先定义 Overview 同步数据结构或配置入口，把天气、课程、待办从外部事实源注入 `xiaoxin_overview_state_t`。
- 接入一个最小真实数据源，例如课程或待办摘要，验证端到端刷新链路。
- 在设备状态卡补充可信电量百分比来源，或继续明确只展示定性状态，避免误导。

建议内容：

- 当前时间、日期、星期。
- 服务端同步的今日天气摘要。
- 服务端或配置端同步的下一节课程摘要。
- 服务端待办系统同步的今日待办数量和摘要。
- 网络、电量、设备温度、可用内存等设备摘要。

设计要点：

- 总览页保持轻量，不做深层复杂操作。
- 数据源通过 Overview state 注入，UI 层只渲染快照。
- 天气由服务端调用 API 后同步到硬件端；课程表由服务端或配置端维护后同步摘要。
- 今日待办优先做服务端事实来源，硬件端第一阶段只显示只读摘要和离线缓存。
- 只显示 3 到 5 个高价值条目，避免小圆屏信息过载。

### P1：宠物情绪系统

当前状态：`部分完成`。

目标：让宠物不只是播放状态动画，而是能根据设备和用户行为表现出“性格”。

已实现：

- 已新增独立 `paopao_pet_mood` 纯逻辑模块，避免把情绪判断散落在 LVGL 显示代码里。
- 服务端 `llm.emotion`、触摸、拖动、摇晃、低电量、WiFi 断开/恢复、语音错误和聊天边缘事件已通过 mood 输入路径转成宠物表现建议。
- BOOT 按键已从宠物 mood 和本地动画触发中释放出来，避免和系统设置入口冲突。
- 已有单元测试和 source-path 测试覆盖情绪冷却、服务端情绪映射、本地互动、BOOT 排除和板级集成路径。
- 宠物核心 GIF 映射已记录到 `docs/xiaoxin-pet-emotion-gif-mapping.zh-CN.md`。

未实现或未完整实现：

- 情绪系统目前更接近“事件到动画建议”，还没有长期能量值、心情值、亲密度或连续性格状态。
- 长时间无交互后的打盹、低功耗宠物状态、充电状态宠物表现还未形成完整策略。
- 宠物短文案、轻量提示和日常陪伴反馈尚未形成独立输出通道。
- 资源仍受 1.46 寸屏、GIF 总大小、PSRAM 和同一时间单动画播放策略约束。

下一步：

- 增加长期状态字段，例如能量、心情和最近互动时间，并让 mood 模块输出可解释的状态快照。
- 把低功耗时钟、充电、长时间无互动和网络恢复纳入一套可验收的宠物表现策略。
- 为新增 GIF 或表情资源建立尺寸、触发、冷却和回退规则。

建议状态维度：

- 能量值：受电量、睡眠、互动频率影响。
- 心情值：受触摸、聊天、摇晃、错误状态影响。
- 专注状态：工作中、思考中、说话中、等待中。
- 异常状态：低电量、断网、识别失败。

可见表现：

- 低电量时宠物变困或焦虑。
- 长时间无交互进入打盹。
- 用户触摸或摇晃后短暂开心。
- 语音识别失败后显示疑惑。
- 网络恢复后显示轻微成功反馈。

设计要点：

- 不把情绪逻辑散落在 UI 代码里，建议增加独立的 `xiaoxin_pet_mood` 或类似模块。
- 情绪模块只输出“建议宠物状态”和“短文案”，显示层负责渲染。
- 情绪变化要有冷却时间，避免动画频繁跳变。

### P2：设置页

当前状态：`部分完成`。

目标：补齐设备本机可配置能力。

已实现：

- BOOT 长按已作为本机设置页主入口，并增加 GPIO0 轮询兜底，提升实机触发可靠性。
- 设置页以 overlay 形式挂载，不加入 Home / Notifications / Overview 三页分页状态机。
- 设置页打开时触摸先交给设置页处理，避免亮度拖动或按钮点击误触发卡片分页。
- 亮度页已从三档预设升级为动态滑条，支持拖动预览、松手持久化和安全亮度下限。
- Wi-Fi 页已复用现有重新配网入口，并修正配网中状态栏图标。
- 关于页已改为面向用户的小芯产品信息。
- 省电项已显示 `已开启` / `已关闭` 状态胶囊，并能同步启停运行时 `PowerSaveTimer`。
- 主页电池图标已在省电开启时显示琥珀色，并保持低电/严重低电颜色优先。
- 已有设置模型 C 测试和设置 source-path 测试覆盖核心交互路径。

未实现或未完整实现：

- 音量、静音、提示音、震动等项目目前仍按目标能力隐藏，尚未接入真实音频/震动配置。
- 宠物主题、动画风格、睡眠时间等偏产品化配置尚未落地。
- 设置页仍以本机小屏高频设置为主，不承担复杂文本输入。
- 省电相关实机体验仍需要在 ESP-IDF 环境中 build / flash 后做完整硬件确认。

下一步：

- 优先补“睡眠/低功耗时间”这类和现有省电调度器直接相关的设置。
- 再评估音量、提示音、震动是否适合当前目标板和音频链路。
- 把复杂输入继续放到 Web 配置面板或服务端配置，不塞进圆屏本机设置页。

入口触发方式：

- BOOT 键长按进入设置界面，作为本机设置的主入口。
- BOOT 键短按继续保留现有聊天/启动阶段配网语义，不与设置入口混用。
- 不再优先使用 Home 页长按、Overview 页底部设置项或电源键短按组合作为设置入口，避免和触摸分页、电源控制职责冲突。

建议功能：

- 亮度调节。
- 音量调节。
- 静音开关。
- WiFi 状态和重新配网。
- 宠物主题或动画风格切换。
- 睡眠时间配置。
- 震动/提示音开关。

设计要点：

- 小圆屏不适合复杂表单，优先使用列表、开关、滑条。
- 每次只展示一个设置项或一组高度相关的设置。
- 设置值写入 NVS，重启后保持。

### P2：Web 配置面板

当前状态：`未开始`。

目标：把不适合在小屏上输入的配置放到浏览器里。

当前基础：

- 项目已有 Wi-Fi 配网页和重新配网入口，可作为后续 Web 配置面板的入口参考。
- 课程、待办、提醒、服务器地址、宠物资源等复杂输入目前尚未形成统一 Web 配置页面。

下一步：

- 先定义最小配置范围：课程/提醒、服务端地址、设备昵称三类即可。
- 复用现有配网页能力，避免在固件端引入过重页面和复杂状态。
- 写入配置后要能通知 Overview、通知中心和设置页刷新。

建议功能：

- 修改设备昵称。
- 配置服务器、MQTT 或 WebSocket 地址。
- 管理提醒或课程表。
- 查看设备状态。
- 上传或切换宠物资源。
- 导出调试信息。

设计要点：

- 复用现有 WiFi 配网页能力。
- 设备端只暴露必要配置接口，避免页面过重。
- 配置写入后通知 UI 刷新，必要时提示重启。

### P2：低功耗与续航体验

当前状态：`部分完成`。

目标：让设备在日常佩戴或桌面使用中更稳定、省电。

已实现：

- 目标板已接入 `PowerSaveTimer(-1, 60, -1)`，进入省电后停留在低功耗时钟页，不再自动关机。
- 低功耗时钟已从简单文本升级为黑底大号时间、动态外圈圆弧和 `POWER 唤醒` 提示。
- 低功耗时钟模型已独立为 `xiaoxin_low_power_clock_model`，并有模型测试和视觉路径测试。
- POWER 短按、BOOT 按键和触摸活动都已接入唤醒/重置省电调度器路径。
- 设置页可启停省电模式，主页电池图标可显示省电状态。
- SNTP 已扩展为三服务器并增加同步回调，提升低功耗时钟时间可靠性。
- 错误提示保持和底部字幕统一流式显示已补齐，减少省电/状态切换时用户看不到错误原因的问题。

未实现或未完整实现：

- 低电量自动降亮度、充电动画、电量曲线校准仍未落地。
- 低功耗页的日期、电量、同步状态等更完整 AOD 信息仍在近期计划中演进。
- 自动息屏、唤醒、低功耗时钟的完整体验还需要真机长时间验证。
- ESP-IDF 完整构建和烧录验证仍需要在对应环境中补跑。

下一步：

- 完成低功耗时钟的日期、电量和同步状态展示，并控制文本刷新频率。
- 增加低电量自动降亮度或省电亮度策略，避免和手动亮度设置冲突。
- 设计充电状态视觉反馈和电量曲线校准流程。

建议功能：

- 自动息屏。
- 触摸、按键、摇晃唤醒。
- 低电量自动降亮度。
- 睡眠模式宠物状态。
- 充电动画。
- 电量曲线校准。

设计要点：

- 低功耗策略要和触摸、动画、网络任务协调。
- 息屏前给出短暂动画或提示，避免用户误以为死机。
- 电量状态变化应同时影响系统浮层、通知中心和宠物情绪。

## 3. 建议实施顺序

### 第一阶段：真实数据接入

当前进度：`部分完成`。通知仓库和低电量/WiFi/语音错误等事件已接入，TTL、OTA 真实流程、课表/提醒真实数据源仍待补齐。

范围：

- 完善真实通知中心。
- 将低电量、WiFi、语音错误、OTA、课程提醒接入通知事件。
- 为通知事件增加 TTL 或状态恢复策略。

收益：

- 现有通知 UI 立即变得有真实用途。
- 改动主要集中在数据层和少量显示同步逻辑，风险较低。

### 第二阶段：总览页动态化

当前进度：`部分完成`。Overview 快照模型、时间日期、网络、电量和空态已落地，天气/课程/待办仍缺真实同步源。

范围：

- 将 Overview 页从静态数组改为可注入数据源。
- 先接入设备状态、时间、提醒，再考虑天气和课程。

收益：

- 用户从 Home 上滑即可看到高频信息。
- 为后续设置页和提醒系统铺路。

### 第三阶段：宠物情绪系统

当前进度：`部分完成`。独立 mood 模块、服务端情绪、本地互动和设备事件映射已落地，长期性格状态和低功耗/充电表现仍待补齐。

范围：

- 增加独立情绪模型。
- 将电量、网络、触摸、摇晃、聊天状态映射到宠物表现。

收益：

- 明显增强 AI Pet 的生命感。
- 让现有动画资源被更充分地使用。

### 第四阶段：设置与配置

### 第五阶段：功耗和稳定性优化

当前进度：`部分完成`。省电调度器、低功耗时钟、POWER 唤醒、SNTP 增强、错误提示保持和运行健康诊断已落地；低电降亮、充电反馈、电量校准和长时间真机验证仍待推进。

关于页显示本次运行、上次运行、最近重启原因和欠压次数。运行时间来自 NVS checkpoint，首次 checkpoint 为 60 秒，之后每 5 分钟保存一次；因此掉电前最后一段运行时间是近似值。

范围：

- 自动息屏、唤醒、低电量策略。
- 运行时内存、FPS、任务负载观察。
- 异常状态恢复和日志导出。

收益：

- 提升实机体验和长期运行稳定性。

## 4. 技术边界建议

- 通知、总览、情绪、设置都应有独立的小模块，避免继续膨胀单个显示文件。
- UI 层尽量只负责渲染和触摸命中，不直接决定业务状态。
- 可测试逻辑优先写成 C/C++ 纯逻辑函数，继续沿用当前本地测试风格。
- 小圆屏信息密度要克制，每页只呈现最关键的信息。
- 动画优先使用轻量 LVGL 属性动画，避免复杂遮罩和实时模糊。

## 5. 风险

### 风险 1：显示文件继续变大

应对：

- 把通知仓库、总览数据、宠物情绪、设置存储拆成独立模块。
- 显示文件只保留对象创建、布局、渲染和触摸分发。

### 风险 2：真实事件和手动清理冲突

应对：

- 状态型事件使用 upsert/remove。
- 用户清理只代表“当前不看”，外部状态再次变化时可以重新出现。

### 风险 3：小屏交互过载

应对：

- 本机只做高频操作。
- 复杂输入放到 Web 配置面板。
- 每个页面保持单一目的。

### 风险 4：动画影响功耗和流畅度

应对：

- 限制同时运行动画数量。
- 页面不可见时暂停无关动画。
- 低电量或息屏前降低刷新频率。

## 6. 下一步建议

最建议优先落地：

1. 给通知中心补 `ttl_ms` 到期清理，并把 OTA 检测/下载/失败真实流程接入 `XIAOXIN_NOTIFICATION_EVENT_OTA_UPDATE`。
2. 基于已落地的 Overview 快照模型，先接入一个最小真实数据源，例如课程或今日待办摘要。
3. 继续推进低功耗时钟近期计划：日期、电量、同步状态展示，以及实机长时间验证。
4. 为宠物情绪系统增加长期状态快照，让能量、心情和最近互动不只停留在一次性事件建议。

这些步骤完成后，小芯会从“局部功能已经可用”继续走向“真实数据、低功耗体验和宠物个性都能稳定闭环”的可用设备。

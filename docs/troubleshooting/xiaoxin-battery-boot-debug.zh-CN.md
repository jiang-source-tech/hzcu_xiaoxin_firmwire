# 小新电池供电启动失败排查记录

## 问题现象

锂电池连接开发板开机时，屏幕背光亮起，但没有任何 UI 界面出现，整个系统无法正常工作。USB 供电时一切正常。

## 硬件背景

- **芯片**：ESP32-S3 @ 240MHz，Octal PSRAM @ 80MHz，16MB QIO Flash
- **板卡**：Waveshare ESP32-S3 Touch LCD 1.46
- **电源管理**：无 PMIC（AXP2101/SY6970），纯 GPIO 控制
  - GPIO7 = 电源保持锁存（输出高电平保持供电）
  - GPIO6 = 电源键输入
  - GPIO8 / ADC1_CH7 = 电池电压检测（3:1 分压）
- **欠压检测**：ESP32-S3 BOD Level 7，阈值约 2.58V

## 排查过程

### 第一阶段：代码审查

通读启动链路的全部代码：

```
app_main()
 → Application::Initialize()
   → Board::GetInstance() → CustomBoard() 构造器
     → InitializePowerHoldEarly()     // GPIO7=1 锁存电源
     → InitializeI2c()                // I2C 总线
     → InitializeTca9554()            // IO 扩展器复位 LCD
     → InitializeSpi()                // QSPI 总线
     → InitializeSpd2010Display()     // 显示面板 + new PaopaoPetDisplay
     → GetBacklight()->RestoreBrightness()  // 背光 75%
     → InitializeTouch() / Buttons() / PowerSaveTimer() / Motion()
   → display->SetupUI()              // LVGL UI 渲染
   → board.StartNetwork()            // WiFi 初始化
```

关键发现：背光在构造器第 6 步就亮了（75%），但 UI 要到 `SetupUI()` 才渲染。如果系统在两者之间复位，用户看到的就是"亮屏无 UI"。

### 第二阶段：添加启动阶段闪烁诊断

由于电池供电时无法使用串口查看日志，采用**背光亮度编码**来诊断启动进度。

最初在构造器末尾加了 3 次闪烁，发现 USB 供电时能看到第 4 次（SetupUI 完成），但电池供电时只有 3 次——说明崩溃发生在构造器内部。

### 第三阶段：逐级亮度定位

给构造器的每个阶段分配不同的背光亮度值，阶段之间用"灭→亮"分隔闪烁：

| 亮度 | 阶段 | 含义 |
|------|------|------|
| 10% | Stage 7 | 背光初始化完成 |
| 18% | Stage 8 | 触摸控制器初始化完成 |
| 26% | Stage 9 | 按钮初始化完成 |
| 34% | Stage 10 | 省电定时器初始化完成 |
| 42% | Stage 11 | 延迟 IMU 初始化完成 |
| 100% | 构造器末尾 | 所有阶段完成 |

通过观察最终停在哪一档亮度，精确定位崩溃位置。

### 第四阶段：定位根因

最终确认：**电池供电时，不加任何间隔地连续初始化所有外设，峰值电流过大导致电池电压跌落到 ESP32-S3 欠压阈值（约 2.58V）以下，触发欠压复位，形成无限启动循环。**

关键因素：
- 240MHz CPU + 80MHz 八线 PSRAM + QSPI 显示屏同时工作
- 背光 75% 占空比额外消耗约 30-50mA
- 各外设初始化之间没有任何延迟，电流峰值叠加
- 无 PMIC 做分级上电，所有外设同时抢电流

## 解决方案

核心思路：**削峰填谷**——降低启动时的峰值电流，并把电流消耗分散到更长的时段。

| 修改 | 效果 |
|------|------|
| 背光从 75% 降到 10% 启动，UI 就绪后再恢复 | 减少约 30-50mA 峰值电流 |
| GPIO7 电源锁存后等待 30ms | 让电池电压从上电冲击中恢复 |
| 各初始化阶段之间加入 150-200ms 间隔 | 避免外设同时抢电流 |
| IMU（QMI8658）初始化延迟 3 秒 | 启动关键窗口再减少约 10mA |
| 提前检测电源类型（电池 vs USB），4500mV 阈值 | USB 供电不做限流，电池供电走省电路径 |
| `GetBatteryLevel()` 根据电源类型正确报告充放电状态 | 电池状态机立即收敛，不再等 2 分钟 |

修改文件：
- `main/boards/waveshare/esp32-s3-touch-lcd-1.46/esp32-s3-touch-lcd-1.46.cc`

## 经验总结

1. **无 PMIC 的电池供电板卡要特别注意启动电流**。没有硬件分级上电，软件必须自己做功率管理。
2. **欠压复位往往表现为"看起来启动了但功能不正常"**，而不是明显的反复重启——因为背光电容或显示面板可能维持短暂的视觉残留。
3. **没有串口时，背光是最可靠的诊断手段**。PWM 亮度编码可以精确定位崩溃点，每个工程师都应该掌握这个技巧。
4. **诊断代码和修复代码应该分开**。诊断用的逐级闪烁在确认修复有效后应该清理掉，只保留真正起作用的省电措施。
